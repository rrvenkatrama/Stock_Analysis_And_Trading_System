Score Formula

positiveCount = Σ (weights > 0)
negativeCount = Σ |weights < 0|
totalSignals = positiveCount + negativeCount

Score = (positiveCount − negativeCount) / max(5, totalSignals) × 100

Range: −100 to +100  ·  Negative scores are valid (strongly bearish)

Key Rules

  • → Denominator floors at 5 — prevents tiny signal sets from inflating scores
  • → Score can go negative — strong bearish signals dominate
  • → Weights are any numeric value — not just ±1
  • → Zero weight = signal ignored entirely

Quick Example

Bullish: +1, +1, +2 = positiveCount 4
Bearish: −1, −1 = negativeCount 2
Total = 6 signals

Score = (4 − 2) / max(5,6) × 100
Score = 2/6 × 100 = 33 → HOLD

Decision Tree

1
Calculate Signal Score

Sum all active bullish signal weights (positive) and all active bearish signal weights (absolute value). Apply formula above.

2
Apply Score-Based Recommendation
score > 50 → BUY
score ≥ 20 → HOLD
score < 20 → SELL
3
Apply Layer 4 Override

Count bearish momentum conditions (see Layer 4 section). If ≥ 3 of 5 are true, force SELL regardless of signal score. This protects against buying into a deteriorating trend.

Layer 4 Momentum Override

Layer 4 is a hard safety valve. Even with strong fundamentals, if the stock is technically breaking down, it forces a SELL recommendation to prevent false entries.

The 5 Conditions

1Price < 50-day Moving Average
250-day MA < 200-day MA (death zone)
3MACD bearish trend
4EMA 9 < EMA 21 (short-term momentum negative)
5SPY < SPY 50-day MA (market regime bearish)
Rule: If 3 or more conditions are true → SELL is forced, regardless of signal score, fundamentals, or analyst consensus.

Buy Execution Flow (9:35 AM ET)

Buying is a two-stage pipeline: rule-based scoring narrows candidates, then Claude Sonnet makes the final ranked picks. Sells are always rule-based — Claude is never involved in exit decisions.

1
Capital Guardrails (checked first)
Cash floor = 20% of total equity ← never spend below this
Spendable = cash − floor
Daily cap = spendable × 25% ← max deployed today
Ticker cap = max 5 new symbols per day

Example: $100K equity, $60K cash → $40K spendable → $10K daily cap. Over 4 bull days, ~68% of available cash is gradually deployed.

2
Candidate Eligibility Filter

Stocks must pass ALL of the following to be considered:

  • Signal score > 50 (recommendation = BUY)
  • Layer 4 bearish count < 3 (momentum not failing)
  • RSI between 30 – 70 (not overbought)
  • Price ≥ $5
  • pick_flag = 1 (manually selected by user)
  • Not already in portfolio (new positions only)
  • Market regime = BULL (SPY above both 50MA and 200MA)
  • No earnings within 5 days
3
Top-10 Candidates Sent to Claude Sonnet

The top-10 eligible stocks (by signal score) are sent to Claude Sonnet with full context for each:

Per-Stock Context
• Score, price, RSI, sector
• All bullish signals + weights
• All bearish signals + weights
• All 5 Layer 4 conditions (✓ ok / ✗ BEARISH)
• Overextension % above 50MA
• Analyst consensus + target upside
Market Context
• Regime: BULL / CAUTION / BEAR
• SPY price, MACD trend, RSI
• VIX level + interpretation
• Fear & Greed index (0–100)
4
Claude Ranks and Selects Picks

Claude considers market sentiment, current trends, and each stock's full signal picture. It returns structured JSON:

{
  "rankings": [ { "rank": 1, "symbol": "AAPL", "buy": true,
    "confidence": "high", "reasoning": "..." }, ... ],
  "symbols_to_buy": ["AAPL", "MSFT"],
  "market_assessment": "Bullish regime with low VIX..."
}
Guardrails: Stocks with Layer 4 ≥ 3 are almost never selected. In high-VIX / fearful markets, Claude prefers defensive names with strong fundamentals. Fallback to top-5-by-score if Claude API fails — autotrader never blocks.
5
Execute in Ranked Order
position_size = min(daily_cap ÷ num_picks, 10% equity) ÷ price
← VIX-scaled: high VIX = smaller size

Claude's rank, confidence level, and reasoning are stored with every trade and visible on the Transactions page. Buying stops when daily cap is exhausted or 5 new tickers have been entered today.

Why Claude for Buys — Not Sells?

Buys: Claude Sonnet ranks candidates using holistic reasoning across signals, market sentiment, and macro context. Humans are bad at picking from 10 similarly-scored stocks — Claude is good at this.
Sells: Always pure rule-based (4-layer algorithm). Hard stops, trailing stops, and momentum deterioration are objective — an LLM could rationalize holding a losing position, which is dangerous for real capital.

How to Change Any Setting

-- Via SQL (immediate effect after next analyzer run)
UPDATE signal_weights SET weight = 3 WHERE signal_name = 'golden_cross_active';
UPDATE signal_weights SET weight = -3 WHERE signal_name = 'death_cross_active';

-- Update autotrader settings (JSON merge)
UPDATE autotrader_settings
SET settings_json = JSON_SET(settings_json, '$.rsi_max', 65)
WHERE category = 'gates';

-- Or use the Settings page in the dashboard (no SQL needed)
