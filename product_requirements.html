1. System Overview

StockTrader is a personal swing trading research and execution system with two independent dashboards sharing the same Alpaca brokerage account, MySQL database, and data layer.

DashboardPortPurpose
Swing Trader3001Short-term scan-and-execute: universe scan, daily plan, human-approved trades
My Stocks8081Personal watchlist research: 113+ stocks, long-hold scoring, dual auto-trading strategies
The My Stocks dashboard (port 8081) is the primary focus of this document and the target for the Auto-Trading feature.
v3.0 addition: The Phoenix Strategy is a second independent auto-trading engine running alongside the existing Alpha Strategy. Both share watchlist data, positions, and the Alpaca account, but operate with separate logic, signals, and configuration.

Rate Limit Management

  • FUND_DELAY = 2000ms between symbols (3 Finnhub calls/symbol run in parallel via Promise.allSettled)
  • QUOTE_DELAY = 800ms between Yahoo quote calls
  • Upgrades: max 60 stale symbols per run, 2000ms delay; weekly cadence
  • Price targets: slow background pass via GET /refresh-targets — 12s/symbol, 15 symbols per run — avoids Yahoo rate limits
  • Yahoo Finance blocks IP after ~10 rapid requests; clears in 30–60 min; switch to Finnhub via sources.js when blocked

7.6 Modals

Why Modal

Triggered by showWhy(sym, why). Splits why string on ' | '. First part (Score:) → large blue heading. Lines starting with + → green. Lines starting with - → red. Neutral → dark text. Close via X button or Escape key.

Buy Modal

Triggered by openBuy(sym, price, name). Fields: Qty, Order Type (market/limit), Limit Price, TIF, Extended Hours. Shows paper/live badge, available buying power, open/closed status. Live cost estimate. Submit → POST /order.

Sell Modal

Mirrors Buy modal. Qty capped at position size. Shows proceeds estimate. Submit → POST /order.

News Modal

Triggered by a "📰 News" button per symbol row. Route: GET /news/:symbol?source=finnhub|sec. Two tabs:

  • Finnhub tab — company news from news_cache table (cached from Finnhub /company-news). Shows headline, source, datetime, summary, and link.
  • SEC EDGAR tab — fetches live from SEC EDGAR Atom RSS: https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&company=SYMBOL&type=8-K&dateb=&owner=include&count=10&output=atom. Shows filing type, date, and link to filing.

Close via X button or Escape key.

Chart Modal (Chart.js 4.4.2 from jsDelivr CDN)

Triggered by openChart(sym, name). Period buttons: 1M/3M/6M/1Y/2Y/5Y. Benchmark toggles: SPY (S&P 500), QQQ (Nasdaq 100), DIA (Dow Jones) — multi-select. All series normalized to 100 at period start. Y-axis shows % change. Data from GET /position-chart/:symbol?period=1y&benchmarks=SPY,QQQ. Chart destroyed on close to prevent memory leaks.

7.7 Data Refresh Pipeline

Runs daily at 8:30 AM ET, Monday–Friday. Manual trigger: GET /refresh-now.

morningJob — 8:30 AM ET Mon–Fri
Phase 1 — Price History (Alpaca, parallel batches of 5)
Phase 2 — Fundamentals (Finnhub primary + Yahoo enrichment, serial 2000ms/symbol)
           └─ getFundamentals() + getAnalystRatings() + getProfile() in parallel via Promise.allSettled
           └─ yf.quote() enrichment (short_float, rec_mean, price targets via COALESCE)
Phase 3 — Upgrades/Downgrades (Finnhub, weekly, max 60 stale symbols, 2000ms delay)
Phase 4 — analyzeAll() → scores all watchlist symbols → stock_signals table
Phase 4.5 — Alpha autoEvaluate(false) — always runs; generates Alpha recommendations for email, never places orders
Phase 4.6 — Phoenix scoreAll() → scores watchlist for Phoenix signals → phoenix_signals table
Phase 5 — scanUniverse() → 204-stock discovery scan
Phase 6 — sendDailyDigest(signals, positions, picks, autoResults, phoenixResults) email
Phase 7 — sendErrorAlert() if any phase collected errors

tradeJob — 9:35 AM ET Mon–Fri
Alpha autoRun() — checks autorun_enabled='1' + market hours → evaluate(true) → live orders → sendAutotraderEmail()
Phoenix autoRun() — checks phoenix_enabled='1' + market hours → Phoenix evaluate(true) → live orders → sendAutotraderEmail()

8. Auto-Trading System

The auto-trading system now contains two independent strategies — Alpha and Phoenix — both layered on top of the existing scoring system. Neither replaces scores, signals, or the dashboard. They run sequentially at 9:35 AM: Alpha executes first, Phoenix executes second.

8.1 Dual Strategy Overview

⚡ Alpha Strategy

File: portfolio_app/autotrader.js

Philosophy: Quality growth timing — buy fundamentally strong stocks at technically optimal entry points.

  • Score 0–100 · BUY ≥50 | HOLD 10–49 | SELL <10
  • 3-tier gate: Regime → Quality → Entry/Exit
  • Hard stop: −8% | Soft exits: score <25, RSI >75, EMA bear cross, MACD bearish, 30d time stop
  • Max 4 positions · 9:35 AM ET execution
  • Toggle: ⚡ Alpha button (blue) in dashboard header

🔥 Phoenix Strategy

Files: portfolio_app/phoenix_screener.js + portfolio_app/phoenix_autotrader.js

Philosophy: Deep value contrarian — buy fundamentally strong stocks that are ≥40% below their 52-week high, waiting for the inevitable recovery.

  • Score 0–100 · BUY ≥60 | WATCH 35–59 | PASS <35
  • 5 hard gates must ALL pass before scoring
  • Hard stop: −15% | Soft exits: score <30, analyst <20% buy, EPS growth negative, recovered within 15% of 52wk high, 60d time stop
  • Max 4 positions · virtual 50% cash pool
  • Toggle: 🔥 Phoenix button (purple) in dashboard header

8.3 Alpha Strategy — Execution Flow

8:30 AM ET — Alpha autoEvaluate(false) → recommendations only, no orders
9:35 AM ET — Alpha autoRun()  [only if autorun_enabled='1' + market hours 9:35–3:45 ET]

8.5 Tier 2 — Quality + Mean Reversion Filter (Alpha)

Universe restriction: The autotrader operates exclusively on stocks in the personal watchlist (watchlist table, is_active=1). Discover universe stocks, manually typed symbols, and any external source are never auto-traded. To make a stock eligible, add it to the watchlist first via "+ Add" or "+ Watch".
Tier 2 — Quality Gate (All Three Must Pass)
  1. Quality gate: Score ≥ 50 AND at least one of: ROE > 15%, EPS growth > 10%, div yield > 1.5%, PE below sector avg. Price ≥ $5 (no penny stocks). asset_type = 'stock' only.
  2. Oversold filter: RSI < 55 AND price not more than 5% above its 50-day SMA.
  3. Volume confirmation: Most recent bar volume ≥ 1.0x 20-day average.

8.6 Tier 1 — Entry/Exit Execution Rules (Alpha)

Tier 1 — Entry/Exit Execution

Entry Conditions (ALL must be met)

  1. Score ≥ 65
  2. At least 2 of: RSI 30–55 | price crossed above 50-SMA ≤3 sessions | MACD bull cross ≤5 sessions | volume ≥1.3x on up day
  3. No existing position in this symbol — if any shares are currently held in Alpaca (regardless of how they were opened), no additional shares are bought
  4. Not at max positions (default 4, configurable in system_config)
  5. Sufficient buying power after 20% cash buffer
  6. Not within 5 days of earnings
  7. Market hours 9:35 AM – 3:45 PM ET
Re-entry rule: Once a position is fully closed (Alpaca qty = 0), the autotrader may repurchase that stock on any future run if entry conditions are met again. There is no permanent blacklist.

8.7 Phoenix Strategy — Overview v3.0 NEW

Phoenix Philosophy: Great companies fall far from their highs for temporary reasons — macro fear, sector rotation, short-term earnings misses. Phoenix finds fundamentally strong stocks ≥40% below their 52-week high that have not yet recovered, waits for the mean reversion, and exits when they approach full recovery.

8.13 Autorun OFF Mode (Recommendation Only)

Default mode. Both strategies generate recommendations without placing orders:

  1. Full refresh runs normally (all phases including Phoenix scoreAll())
  2. Both autotraders evaluate logic but do NOT place orders
  3. Daily digest gains:
    • "Alpha Recommendations (Autorun OFF)" — BUY/SELL with scores and reasons
    • "Phoenix Recommendations (Phoenix OFF)" — BUY/WATCH with deep-value thesis
    • "⭐ Confluence Signals" — any symbol where both = BUY

8.14 Autorun ON Mode (Auto-Execute)

At 9:35 AM ET, strategies execute in sequence:

  1. Alpha autoRun() executes first — regime check → exits → entries → sendAutotraderEmail()
  2. Phoenix autoRun() executes second — hard gates → exits → entries → sendAutotraderEmail()
  3. Both strategies operate independently — Alpha selling a stock does not prevent Phoenix from buying it (they track positions via strategy column)
  4. All orders recorded in autotrader_trades with the appropriate strategy value
  5. All decisions logged to system_log (source = 'autotrader' or 'phoenix')
Concurrency guard: autorunning and phoenixRunning flags prevent multiple simultaneous runs per strategy.

10. Known Issues & Constraints

  1. Finnhub /stock/price-target returns 403 (paid). Workaround: use Yahoo Finance yf.quote() fields targetMeanPrice/High/Low via /refresh-targets slow-fetch (12s/symbol, 15/run).
  2. VIXY proxy for VIX is approximate — VIXY × 1.8 + 2 ≈ VIX. Acceptable for regime filter.
  3. Yahoo Finance rate-limits after ~10 rapid requests — clears in 30–60 min. /refresh-targets handles this at 12s/symbol to stay under the limit.
  4. MySQL 8.4: no ADD COLUMN IF NOT EXISTS — migration scripts must use information_schema checks before adding columns. See db/migrate_phoenix.sql for pattern.
  5. VIXY and SPY must be in watchlist — Alpha's Tier 3 regime filter and VIX sizing require SPY and VIXY price_history rows to be populated.
  6. shares_buyback_pct always null — Finnhub free tier does not provide buyback data. Phoenix buyback scoring (+6) is always 0 until manually populated in watchlist table.
  7. price_history stores ~380 days only — 2Y/5Y gains show "—" in table. Chart modal fetches from Alpaca on-demand.
  8. No intraday monitoring — both autotraders run once at 9:35 AM. Stop-loss checked next day. Intentional (swing, not day trading).
  9. Alpaca paper mode default — set ALPACA_BASE_URL in .env to go live.
  10. MySQL JSON column gotcha — plan_json and reasons returned as already-parsed objects by mysql2. Always guard: typeof row.col === 'string' ? JSON.parse(row.col) : row.col.

11. Design Decisions (Do Not Reverse Without Discussion)

  1. Human control always available — both Autorun OFF modes are the default and must remain functional. Each toggle reachable in one click from header.
  2. Plain <a href> for navigation-critical actions — Toggle Autorun/Phoenix, Scan Now, Refresh Now, Add/Remove ticker. No JS onclick. Learned from production failures.
  3. No day trading — Minimum hold 1 trading day. Alpha time stop at 30 days; Phoenix at 60 days. Target: 5–20 day Alpha holds, 20–60 day Phoenix holds.
  4. No margin, no borrowing — instant settlement only. 20% cash buffer (Alpha) + 50% virtual pool (Phoenix) always enforced.
  5. Exit-first philosophy — in all autotrader.run() calls, exits evaluated before entries. Freeing capital takes priority.
  6. Alpha: all 3 tiers must agree — one tier veto blocks the trade. No partial overrides.
  7. Phoenix: all 5 gates must pass — any gate failure means PASS. No score averaging over bad fundamentals.
  8. Scoring engine unchanged by autotrader — both autotraders are consumers of their respective signal tables. Existing scoring runs unchanged; autotraders add gating logic on top.
  9. Anti-churn — Alpha score drift (60→45) does NOT trigger a sell. Only sell at score <25 or hard exit conditions.
  10. Wider hard stop for Phoenix — deep value stocks need more room to be right. −15% vs Alpha's −8%. Phoenix is a longer hold with more volatility tolerance.
  11. Wider time stop for Phoenix — 60 days vs Alpha's 30 days. Mean reversion takes longer than technical momentum plays.
  12. Strategies share account, not position logic — both use the same Alpaca account and watchlist, but track positions independently via the strategy column. A stock can theoretically be held by both simultaneously.
  13. Confluence is informational only — when both strategies flag BUY, it highlights conviction. It does NOT trigger a separate order or any special execution path.
  14. Paper mode safe for autorun testing — both strategies in paper mode place paper orders, not live. Validate behavior before switching to live.
  15. Watchlist is the autotrader's universe — adding a stock to the watchlist is the explicit human act of authorising the autotrader (either strategy) to consider it.
  16. One position per symbol per strategy, no averaging down — Alpha will not add to an existing Alpha position; Phoenix will not add to an existing Phoenix position.
  17. Re-entry is permitted — once fully out (qty = 0 for that strategy), a stock can be repurchased. No permanent blacklists.
  18. Manual positions inherit Alpha exit rules on Autorun ON — Alpha evaluates all Alpaca positions including manually opened ones. Phoenix evaluates only positions in autotrader_trades with strategy='phoenix'.
  19. No immediate execution on toggle — turning either strategy ON mid-day triggers no immediate orders.
